Getting Started

Our API allows you to generate AI-powered images programmatically. To get started:

  1. Obtain your API key from the dashboard
  2. Include the API key in the Authorization header
  3. Make requests to our API endpoints

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

Authentication

Authenticate your API requests by including your secret API key in the Authorization header.

curl https://api.kenthir.my.id/v1/images/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"a beautiful sunset"}'

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

Generate Image

POST https://api.kenthir.my.id/v1/images/generate

Creates an image based on a text prompt. Returns the generated image URL or binary data.

Request Parameters

Parameter Type Required Description
prompt string Yes A text description of the desired image(s). Maximum 1000 characters.
model string No ID of the model to use. Defaults to flux.
size string No The size of the generated images. Must be one of 1024x1024, 1024x1792, or 1792x1024. Defaults to 1024x1024.
quality string No The quality of the image. Must be one of standard, hd, or ultra. Defaults to standard.
n integer No The number of images to generate. Must be between 1 and 4. Defaults to 1.
seed integer No A seed for deterministic generation. If not specified, a random seed will be used.

Response

The API returns a JSON object containing the generated image URLs and metadata.

Field Type Description
created integer The Unix timestamp (in seconds) when the image was generated.
data array An array of generated images.
data[].url string The URL of the generated image. Expires after 24 hours.
data[].seed integer The seed used for generation.

Rate Limits

We apply rate limits to ensure fair usage of our API:

  • Free tier: 10 requests per minute, 1000 requests per month
  • Pro tier: 30 requests per minute, 10,000 requests per month
  • Enterprise tier: 60 requests per minute, unlimited requests

If you exceed these limits, you'll receive a 429 Too Many Requests response. Check the following response headers for information about your current rate limit status:

Header Description
X-RateLimit-Limit The maximum number of requests you're permitted to make per minute.
X-RateLimit-Remaining The number of requests remaining in the current rate limit window.
X-RateLimit-Reset The time at which the current rate limit window resets in UTC epoch seconds.

Best Practices

Prompt Engineering

  • Be specific and descriptive in your prompts
  • Include style references (e.g., "digital art", "photorealistic")
  • Specify composition and framing when important
  • Use commas to separate different aspects of your prompt

Error Handling

  • Always check for errors in the response
  • Implement retry logic with exponential backoff
  • Cache successful responses when appropriate
  • Handle rate limits gracefully

Performance

  • Use webhooks for long-running generation tasks
  • Consider pre-generating common images
  • Implement client-side caching of images
  • Use the appropriate quality setting for your use case
