Pahami Dulu Konsep Dasarnya

Biar gak bingung saat melakukan pengaturan & troubleshooting.

Domain

Alamat unik website seperti namakamu.my.id. Ibarat alamat rumah di dunia digital. Manusia mengingat nama, komputer membutuhkan deretan angka (IP Address).

DNS

Domain Name System = buku telepon internet. Tugasnya menerjemahkan domain menjadi IP address. Tanpanya, kita harus hafal deretan angka seperti 185.199.108.153.

Propagasi DNS

Waktu sinkronisasi (bisa memakan waktu 1-24 jam) setelah mengubah pengaturan NameServer atau DNS. Seluruh server internet butuh waktu memperbarui data. Sabar ya!

Bab 1: Buat Web dengan AI

Bikin website statis tanpa perlu jago coding — cukup berikan instruksi ke Google AI Studio!

Contoh Prompt:

"Buat website portofolio untuk fotografer dengan tema gelap, foto profil melingkar, daftar layanan, dan tombol WhatsApp."

1

Buka Google AI Studio

Kunjungi aistudio.google.com dan login menggunakan akun Google Anda.

2

Pilih Mode Web

Pastikan Anda memilih template atau mode yang mendukung output kode HTML/CSS/JS.

3

Berikan Instruksi (Prompt)

Ketik prompt sesuai dengan imajinasi Anda. Jangan ragu untuk meminta AI merevisi desain jika ada yang kurang cocok.

4

Download Kode

Salin kodenya dan simpan ke dalam satu file komputer Anda dengan nama persis index.html.

File index.html adalah pintu masuk utama website Anda yang akan dibaca pertama kali oleh browser.

Bab 2: Beli Domain .my.id

Identitas website Anda di internet, contoh menggunakan layanan IDwebhost.

01

Kunjungi idwebhost.com atau penyedia layanan domain lokal lainnya.

02

Cari nama domain idaman Anda di kolom pencarian dengan akhiran .my.id.

03

Jika berstatus Tersedia (warna hijau), klik Beli.

04

Isi detail akun dan KTP (Domain .id wajib menggunakan KTP sesuai regulasi PANDI).

05

Lakukan pembayaran. Domain Anda akan aktif dalam beberapa menit!

.my.id

Domain paling pas untuk portofolio personal, UMKM, dan pelajar. Murah, unik, dan menunjukkan identitas Indonesia.

Bab 3: GitHub Repo

Tempat menyimpan kode website agar bisa diakses internet 24/7 secara gratis.

  • Daftar dan login di github.com
  • Klik "New" untuk membuat repository baru. Beri nama bebas (misal: web-portofolio).
  • Pastikan visibilitas diset ke Public.
  • Di halaman repo, klik Add file → Upload files. Drag & drop file index.html yang Anda buat di Bab 1.
  • Klik tombol hijau Commit changes.

Bab 4: Deploy (Online-kan)

  1. Di dalam Repo GitHub Anda, masuk ke tab Settings.
  2. Lihat menu kiri, klik Pages.
  3. Pada bagian Source, pilih Deploy from a branch.
  4. Di dropdown branch, pilih main lalu klik Save.
  5. Tunggu 1-2 menit lalu refresh halaman. Akan muncul URL website sementara Anda.

Website Anda kini online di:

https://username.github.io/nama-repo/

Bab 5: Integrasi Cloudflare

Hubungkan domain .my.id dengan GitHub Pages, sekaligus mengamankan website dengan SSL Gratis dari Cloudflare.

PERHATIAN PENTING (Sering Salah)

Ikuti urutan langkah di bawah ini dengan persis. Jangan aktifkan Proxy (Awan Orange) di Cloudflare sebelum GitHub Pages berhasil menerbitkan sertifikat SSL.

1 Daftar & Arahkan NameServer

Buat akun di cloudflare.com. Klik "Add site", masukkan domain namakamu.my.id, dan pilih paket Free.

Cloudflare akan memberi 2 NameServer (misal: elias.ns.cloudflare.com). Login ke IDwebhost, masuk ke pengaturan domain Anda, dan ubah NameServer bawaan menjadi NameServer dari Cloudflare.

2 Tambahkan DNS Record

Di dashboard Cloudflare, buka menu DNS. Masukkan 4 record IP GitHub Pages dan 1 CNAME.

Type A     Name: @    Value: 185.199.108.153
Type A     Name: @    Value: 185.199.109.153
Type A     Name: @    Value: 185.199.110.153
Type A     Name: @    Value: 185.199.111.153
Type CNAME Name: www  Target: username.github.io

PENTING: Matikan Proxy status (Awan harus abu-abu / DNS Only) untuk semua record di atas!

3 Setting di GitHub Pages

Kembali ke GitHub Repo > Settings > Pages. Di kolom Custom domain, ketikkan domain Anda (namakamu.my.id) lalu Save.

Tunggu proses verifikasi DNS (butuh beberapa menit hingga jam jika DNS belum menyebar). Jika sudah berhasil, centang "Enforce HTTPS".

4 Nyalakan Proteksi Penuh

Setelah website berhasil diakses via HTTPS secara normal, kembali ke Cloudflare.

1. Masuk ke menu DNS > Edit record sebelumnya > Nyalakan kembali Proxy (Awan Orange).

2. Masuk ke menu SSL/TLS > Overview > Pilih mode Full (strict).

Tips Saat Menghadapi Masalah

Website tidak muncul / Error 404 Pastikan file bernama tepat index.html (huruf kecil semua) dan berada di bagian root terluar repository, bukan di dalam folder lain.
Error 525/526 saat pasang domain GitHub belum menerbitkan sertifikat SSL. Matikan fitur Proxy (Awan Orange jadi Awan Abu-abu) di Cloudflare sementara waktu, lalu cek kembali GitHub Pages.
Tampilan rusak / CSS tidak meload Jika Anda menggunakan asset eksternal (gambar/css), pastikan menggunakan *Relative Path* (seperti ./gambar.jpg) atau masukkan ke folder repository.
Domain sudah dipakai (IDwebhost) Nama domain bersifat *First Come, First Served*. Jika sudah diambil orang lain, coba gunakan angka, singkatan, atau kombinasi nama lain.
