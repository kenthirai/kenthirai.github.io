Bab 00

Pahami Dulu Konsep Dasarnya

Biar gak bingung saat melakukan pengaturan & troubleshooting.

Domain

Alamat unik website seperti namakamu.my.id. Ibarat alamat rumah di dunia digital agar mudah diingat manusia.

DNS

Domain Name System = buku telepon internet. Tugasnya menerjemahkan nama domain menjadi IP address angka yang dimengerti server.

Propagasi

Waktu sinkronisasi (bisa 1-24 jam) setelah pengaturan DNS diubah. Seluruh server internet butuh waktu memperbarui data.

Static Site

Website yang hanya berisi file HTML, CSS, dan JS. Sangat ringan, cepat, dan punya tingkat keamanan tinggi karena tanpa database server.

SSL (HTTPS)

Protokol keamanan yang mengenkripsi data antara browser dan website. Membuat website Anda dipercaya browser (ikon gembok hijau).

CDN

Content Delivery Network. Jaringan server global (seperti Cloudflare) yang menyimpan salinan web Anda agar loading cepat di mana saja.

Bab 03

GitHub Repo

Tempat menyimpan kode website agar bisa diakses internet 24/7 secara gratis dengan performa tinggi.

  • Daftar dan login di github.com menggunakan email aktif.
  • Klik "New" untuk membuat repository baru. Beri nama bebas (misal: web-portofolio).
  • Pastikan visibilitas diset ke Public agar fitur GitHub Pages bisa aktif secara gratis.
  • Klik "uploading an existing file". Drag & drop file index.html (dan file pendukung lainnya seperti style.css) ke kotak yang tersedia.
  • Tulis pesan singkat di kolom "Commit changes" (misal: "Initial web upload") lalu klik tombol hijau.
Commit

Setiap kali Anda mengubah kode dan menyimpannya, itu disebut "Commit". GitHub merekam sejarah setiap perubahan yang Anda lakukan.

Branch

Cabang dari kode Anda. Secara default kita menggunakan main sebagai cabang utama yang akan ditampilkan online.

Langkah Deploy

Online-kan Website

  1. 1. Masuk ke tab Settings di navigasi atas Repo Anda.
  2. 2. Klik menu Pages di bagian "Code and automation" (sisi kiri).
  3. 3. Pada bagian Build and deployment, pilih Deploy from a branch.
  4. 4. Pilih branch main, pilih folder / (root), lalu klik Save.

Live URL Preview:

https://username.github.io/nama-repo/

Cara Update Tampilan Web:

Ingin mengubah desain atau teks? Cukup edit file di komputer Anda, lalu upload ulang file yang baru ke GitHub Repo Anda dengan cara yang sama seperti di langkah awal. GitHub akan otomatis melakukan "re-deploy" dalam beberapa detik!

Bab 05

Integrasi Cloudflare

Hubungkan domain .my.id dengan GitHub Pages, sekaligus mengamankan website dengan SSL Gratis dari Cloudflare.

Perhatian Penting (Sering Salah)

Ikuti urutan langkah di bawah ini dengan persis. Jangan aktifkan Proxy (Awan Orange) di Cloudflare sebelum GitHub Pages berhasil menerbitkan sertifikat SSL.

1

Arahkan NameServer

Cloudflare bertindak sebagai "polisi lalu lintas" di depan domain Anda. Untuk mengaktifkannya, domain harus "dikendalikan" oleh Cloudflare.

Ganti NameServer di IDwebhost (atau registrar lain) menjadi:

  • dexter.ns.cloudflare.com
  • mimi.ns.cloudflare.com

*Nama NS mungkin berbeda untuk setiap akun.

2

Input DNS Wajib

Masukkan 5 record ini agar domain tersambung ke server GitHub. Gunakan @ untuk domain utama.

TYPENAMECONTENT / VALUE
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwusername.github.io

STATUS: Pastikan Proxy diset ke "DNS Only" (Awan Abu-abu). Jangan "Proxied" (Awan Orange) dulu!

3

Sertifikat SSL GitHub

GitHub perlu menerbitkan sertifikat SSL khusus untuk domain Anda agar bisa diakses via https://.

  • Masukkan domain di GitHub > Settings > Pages.
  • Tunggu sampai muncul tulisan "DNS check successful".
  • Centang "Enforce HTTPS". Selesai!
4

Aktifkan Fitur Sultan

Sekarang website sudah online. Aktifkan kembali Proxy (Awan Orange) untuk menikmati fitur premium Cloudflare.

Caching: Web Anda jadi 10x lebih cepat.
WAF: Blokir bot jahat dan serangan DDoS.
Analytics: Pantau berapa banyak pengunjung tiap hari.

Kenapa Awan Harus Abu-abu (DNS Only)?

GitHub Pages perlu melakukan "kontak langsung" dengan domain Anda untuk memverifikasi kepemilikan dan menerbitkan sertifikat SSL (Let's Encrypt).

Jika Proxy aktif (Awan Orange), Cloudflare akan menutupi server asli Anda. GitHub tidak bisa melihat record DNS Anda, sehingga verifikasi akan gagal terus menerus (stuck di "Waiting for DNS").

Troubleshooting Umum

Error 525 / 526 SSL Handshake?

Pastikan mode SSL/TLS di Cloudflare diset ke Full (Strict). Error ini terjadi karena GitHub menggunakan sertifikat SSL sendiri, dan Cloudflare bingung cara menyambungnya jika diset ke mode lain.

Domain belum aktif?

Proses perambatan DNS (Propagasi) butuh waktu 5 menit hingga 24 jam. Sabar adalah kunci!

Advanced

Workers vs Pages: Apa Bedanya?

Seringkali pemula bingung memilih dua fitur utama ini. Mari kita bedah perbedaannya.

Cloudflare Pages

Dirancang khusus untuk Hosting Website. Sangat mirip dengan GitHub Pages, namun dengan integrasi CDN Cloudflare yang lebih mendalam.

Kegunaan:
  • Hosting Landing Page / Portofolio
  • Web Statis (HTML, CSS, JS)
  • Framework (Next.js, Astro, React)

Cloudflare Workers

Platform Serverless Code. Tempat untuk menjalankan fungsi logic di "Edge" (server terdekat dengan user) tanpa perlu menyewa server VPS.

Kegunaan:
  • Membuat API / Backend Ringan
  • Redirect link secara custom
  • Modifikasi request/response HTTP
Fitur Cloudflare Pages Cloudflare Workers
Fokus Utama Konten / File Website Logika / Kode Program
Cara Pakai Konek ke GitHub / Upload Folder Tulis Script (JS/Rust)
Kemudahan Sangat Mudah (Tanpa Coding) Menengah (Butuh Coding)
Analogi Gudang penyimpanan barang Robot yang mengerjakan tugas

Cara Menggunakan Pages:

Jika Anda ingin memindahkan website dari GitHub Pages ke Cloudflare secara penuh:

  1. Buka dashboard Cloudflare > menu Build > Compute > Workers & Pages.
  2. Klik Create application > Tab Pages > Connect to Git.
  3. Otorisasi GitHub: Anda akan diarahkan ke GitHub. Jika baru pertama kali, Anda harus menginstal aplikasi "Cloudflare Workers and Pages".
  4. Tips: Pilih "All repositories" agar semua repo masa depan otomatis terbaca, atau "Only select repositories" jika ingin memilih repo tertentu saja.
  5. Setelah aplikasi terinstal, pilih Repository website Anda dari daftar yang muncul di Cloudflare.
  6. Build Settings: Untuk website statis HTML, pilih Framework preset "None" dan biarkan Build command kosong.
  7. Klik Save and Deploy. Cloudflare akan membaca file Anda dan memberitahukan jika proses "Success".

Cara Menggunakan Workers:

Jika Anda ingin membuat fitur dinamis (misal: redirect atau API):

  1. Buka dashboard Cloudflare > menu Build > Compute > Workers & Pages.
  2. Klik Create application > Tab Workers.
  3. Klik Create Worker dan mulai tulis kode JavaScript Anda.

Q&A

Pertanyaan Sering Diajukan

Semua yang perlu Anda ketahui tentang membangun website mandiri.

Apakah tutorial ini benar-benar gratis?

Ya! Semua platform yang direkomendasikan (GitHub, Cloudflare, dan Alat AI) memiliki Free Tier yang sangat mumpuni untuk website skala kecil hingga menengah tanpa biaya bulanan.

Berapa lama sampai website saya online?

Proses upload kode hanya hitungan menit. Namun, untuk domain baru (seperti .my.id), diperlukan waktu sinkronisasi (propagasi) DNS sekitar 1-24 jam sebelum bisa diakses seluruh dunia.

Apakah saya harus paham bahasa pemrograman?

Tidak wajib. Berkat bantuan AI, Anda cukup memberikan instruksi bahasa manusia, dan AI akan menuliskan kodenya. Tutorial ini hanya mengajarkan cara "merakit" hasil tersebut agar online.

Bisa gunakan domain selain .my.id?

Tentu saja. Langkah integrasi Cloudflare dan GitHub sama persis baik Anda menggunakan domain .com, .id, .net, atau ekstensi lainnya.

Bagaimana cara mengupdate isi website?

Cukup edit file index.html di komputer Anda, lalu upload ulang file tersebut ke Repository GitHub yang sama. Website Anda akan otomatis terupdate dalam hitungan detik.

Troubleshooting Tips

Website tidak muncul / Error 404

Pastikan file bernama tepat index.html (huruf kecil semua) dan berada di root terluar repository, bukan di dalam folder lain.

Error 525/526 saat pasang domain

GitHub belum menerbitkan sertifikat SSL. Matikan fitur Proxy (Awan Abu-abu) di Cloudflare sementara waktu, lalu cek kembali GitHub Pages.

Tampilan rusak / CSS tidak meload

Jika menggunakan asset eksternal, pastikan menggunakan *Relative Path* (seperti ./style.css) agar terpanggil dengan benar.

Domain sudah dipakai (IDwebhost)

Nama domain bersifat *First Come, First Served*. Jika sudah diambil, coba gunakan angka, singkatan, atau kombinasi nama kreatif lainnya.
